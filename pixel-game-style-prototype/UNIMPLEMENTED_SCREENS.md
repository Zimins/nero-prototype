# Pixel Game Style Prototype - 미구현 화면 분석

## 프로젝트 개요

픽셀 아트 스타일의 쥬크박스 프로토타입으로, 다양한 방(Room)을 탐험하며 음악을 듣고 소통할 수 있는 인터랙티브 웹 앱입니다.

---

## 현재 구현 상태

### ✅ 구현 완료된 화면/기능

| 화면 | 파일 | 구현 상태 | 설명 |
|------|------|----------|------|
| 거실 | `LivingRoom.tsx` | ✅ 완료 | 배경 이미지, 벽난로(채팅), 사진첩(방명록) 인터랙션 |
| 다락방 | `AtticRoom.tsx` | ⚠️ 부분 | 배경 이미지만 표시 (인터랙션 없음) |
| 상점 | `ShopPage.tsx` | ✅ 완료 | 플레이리스트 구매 UI, 코인 시스템 |
| 방 꾸미기 | `DecorPage.tsx` | ✅ 완료 | 아이템 카테고리, 장착 기능 |
| 쥬크박스 | `JukeboxPopup.tsx` | ✅ 완료 | 드래그 가능, 재생 컨트롤, 플레이리스트 |
| 채팅 | `ChatPopup.tsx` | ✅ 완료 | 메시지 송수신 UI |
| 방명록 | `GuestbookPopup.tsx` | ✅ 완료 | 방명록 작성/조회 |
| 하단 네비게이션 | `BottomNavigation.tsx` | ✅ 완료 | 방 이동, 자동 숨기기 설정 |

## 공통 필요 리소스

### 픽셀 아트 이미지
- 서재 배경
- 도서관 배경
- 천문대 배경 (밤하늘)
- 온실 배경
- 다실 배경
- 외부 환경 배경

### 공통 컴포넌트 패턴
모든 방 컴포넌트는 `LivingRoom.tsx`의 패턴을 따르면 됨:
```tsx
// 기본 구조
export function RoomName() {
  const [hoveredItem, setHoveredItem] = useState<string | null>(null);

  return (
    <div className="relative w-full h-full flex items-center justify-center">
      <div className="relative">
        <ImageWithFallback
          src={backgroundImage}
          alt="방 이름"
          className="max-w-full max-h-[calc(100vh-200px)] object-contain pixel-render"
        />

        {/* 인터랙티브 영역 */}
        <button className="absolute ...">
          {/* 클릭 가능 영역 */}
        </button>

        {/* 호버 라벨 */}
        {hoveredItem === 'item' && (
          <div className="absolute ...">라벨</div>
        )}
      </div>
    </div>
  );
}
```
